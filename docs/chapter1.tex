\subsection{Анализ программных инструментов разработки веб-приложений}

Для реализации проекта выбран подход создания веб-приложения на базе языка программирования Python. Этот выбор обусловлен высокой скоростью разработки, читаемостью кода и богатой экосистемой библиотек.

\begin{itemize}
    \item \textbf{Frontend-часть:} Пользовательский интерфейс будет реализован с использованием классических веб-технологий (HTML5, CSS3, JavaScript). Для рендеринга динамического контента на стороне сервера выбран шаблонизатор \textbf{Jinja2}, тесно интегрированный во Flask.
    \item \textbf{Backend-фреймворк:} В качестве основы серверной части выбран микрофреймворк \textbf{Flask}. В отличие от тяжеловесных решений (таких как Django), Flask предоставляет гибкость архитектуры, позволяя разработчику самостоятельно выбирать необходимые модули. Он отлично подходит для реализации маршрутизации, обработки HTTP-запросов, а также интеграции подсистем аутентификации (например, через Flask-Login).
    \item \textbf{Работа с базой данных:} Для взаимодействия с базой данных используется \textbf{SQLAlchemy} — мощная библиотека объектно-реляционного отображения (ORM). Использование ORM позволяет описывать структуру таблиц (Пользователи, Проекты, Задачи) в виде классов Python, избегая написания чистых SQL-запросов. SQLAlchemy обеспечивает безопасность (защиту от SQL-инъекций), поддержку CRUD-операций и удобные инструменты для агрегации данных (расчета статистики по проектам).
\end{itemize}

\subsection{Формулировка цели и задач работы}

На основе проведенного анализа предметной области (п. 1.1) и выбора технологического стека (п. 1.2), главная \textbf{цель работы} заключается в создании веб-приложения для управления проектами и отслеживания задач на базе фреймворка Flask.

Для достижения поставленной цели необходимо решить следующие \textbf{задачи}:
\begin{enumerate}
    \item Провести анализ целевой аудитории и спроектировать пользовательские сценарии (Use Cases);
    \item Спроектировать логическую и физическую модели реляционной базы данных;
    \item Разработать макеты пользовательского интерфейса (Wireframes);
    \item Реализовать подсистему аутентификации пользователей и настроить систему распределения ролей;
    \item Разработать программные модули (CRUD-интерфейсы) для управления сущностями «Проект», «Задача» и «Пользователь»;
    \item Реализовать функционал загрузки и скачивания файлов, прикрепленных к задачам;
    \item Разработать модуль агрегирования данных для вывода аналитической статистики по проектам;
    \item Развернуть готовое веб-приложение на удаленном сервере (хостинге).
\end{enumerate}
